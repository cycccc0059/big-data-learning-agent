# 大数据面试准备

## 常见面试方向

### 技术基础
- SQL 能力（窗口函数、复杂 join、执行计划分析）
- 编程能力（Java/Scala/Python）
- 数据结构与算法

### 组件深度
- Spark：shuffle、内存管理、调优、Core vs SQL
- Flink：checkpoint、状态后端、反压、两阶段提交
- Hive：分区分桶、优化、与 Spark 对比
- Kafka：可靠性、ISR、exactly-once、高性能设计

### 数仓设计
- 分层方法（ODS、DWD、DWS、ADS）
- 维度建模（星型、雪花模型）
- 数据质量、血缘、指标体系建设

### 项目经验
- 准备 2-3 个有深度的项目案例
- STAR 法则：背景、目标、行动、结果
- 关键数字：数据量、性能提升、成本节约

### 系统设计
- 实时数仓架构设计
- 埋点 + Kafka + Flink + 存储 + 服务的实时链路
- 调度与监控体系

## 回答框架

每个面试题建议用以下结构回答：
1. **一句话结论**
2. **核心机制**（为什么这样设计）
3. **结合实际**（你用过/见过的场景）
4. **延伸思考**（优化方向、替代方案、权衡）
