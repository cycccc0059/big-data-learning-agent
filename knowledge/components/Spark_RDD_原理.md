# Spark RDD 原理笔记

## 1. 核心概念

### 1.1 RDD 定义
**RDD（Resilient Distributed Dataset）** 是 Spark 的核心抽象，代表一个**不可变、分区、可并行计算**的分布式数据集合。

### 1.2 RDD 五大核心属性
每个 RDD 内部由以下五个属性定义：

| 属性 | 说明 |
|------|------|
| **分区列表** | 数据被划分成的分片集合 |
| **分区计算函数** | 对每个分区执行的计算逻辑 |
| **依赖关系列表** | 对父 RDD 的依赖（窄依赖/宽依赖） |
| **分区器（可选）** | 键值对 RDD 的哈希分区器等 |
| **优先位置列表（可选）** | 数据本地性信息（如 HDFS 块位置） |

### 1.3 创建方式
- **并行化集合**：从 Driver 端的 Scala/Python 集合创建
- **外部数据源**：从 HDFS、HBase、Cassandra 等读取
- **转换操作**：从已有 RDD 通过 `map`、`filter` 等转换生成

### 1.4 共享变量
- **广播变量（Broadcast Variables）**：在所有节点缓存只读变量，避免重复传输
- **累加器（Accumulators）**：仅支持“添加”操作的变量，用于计数、求和等

## 2. 关键机制

### 2.1 分区与并行度
- 数据被自动或手动划分为多个分区
- 每个分区对应一个 Task，在集群节点上并行执行
- 分区数决定并行度，可通过 `repartition`/`coalesce` 调整

### 2.2 依赖类型
| 依赖类型 | 特点 | 示例 |
|---------|------|------|
| **窄依赖** | 每个父分区最多被子分区使用一次 | `map`、`filter`、`union` |
| **宽依赖** | 每个父分区可能被子分区多次使用 | `groupByKey`、`reduceByKey`、`join` |

### 2.3 Shuffle 机制
- **触发条件**：宽依赖操作（如 `groupByKey`、`reduceByKey`）
- **性能影响**：涉及磁盘 I/O、网络传输、序列化/反序列化，是性能瓶颈
- **优化方向**：减少 Shuffle 数据量、避免不必要的 Shuffle

### 2.4 持久化（Persistence）
- **作用**：复用中间结果，避免重复计算
- **存储级别**：

| 级别 | 存储位置 | 特点 |
|------|---------|------|
| `MEMORY_ONLY` | 内存 | 速度快，内存不足时重新计算 |
| `MEMORY_AND_DISK` | 内存+磁盘 | 内存不足时溢写到磁盘 |
| `DISK_ONLY` | 磁盘 | 适用于数据量大、内存不足 |
| `MEMORY_ONLY_SER` | 序列化内存 | 节省内存，增加 CPU 开销 |
| `OFF_HEAP` | 堆外内存 | 减少 GC 压力 |

### 2.5 容错机制
- **血统（Lineage）**：记录 RDD 的转换依赖链
- **故障恢复**：丢失的分区通过血统重新计算，无需全局重算
- **Checkpoint**：将血统截断，保存到可靠存储（如 HDFS），适用于长血统场景

## 3. 常见问题与优化

### 3.1 性能问题
| 问题 | 原因 | 优化方案 |
|------|------|---------|
| **数据倾斜** | 分区数据分布不均 | 加盐、调整分区器、增大并行度 |
| **Shuffle 过多** | 频繁使用宽依赖操作 | 使用 `reduceByKey` 替代 `groupByKey`、预聚合 |
| **内存溢出** | 单个分区数据过大 | 增大分区数、使用 `MEMORY_AND_DISK` 持久化 |
| **小文件问题** | 输入数据大量小文件 | 合并小文件、使用 `wholeTextFiles` |

### 3.2 常见误区
- **闭包陷阱**：在算子内部引用外部变量时，变量会被序列化到每个 Task，可能导致意外行为
- **重复计算**：未对多次使用的 RDD 进行持久化，导致重复执行血统
- **不合理的分区数**：分区数过少导致并行度不足，过多导致调度开销

### 3.3 优化建议
1. **合理设置并行度**：一般设为 CPU 核心数的 2-3 倍
2. **优先使用窄依赖**：减少 Shuffle 操作
3. **使用 `reduceByKey` 替代 `groupByKey`**：在 Shuffle 前进行本地聚合
4. **对复用 RDD 进行持久化**：选择合适的存储级别
5. **利用数据本地性**：确保计算靠近数据存储位置

## 4. 学习建议

### 4.1 学习路径
1. **理解基础概念**：RDD 定义、分区、依赖、血统
2. **掌握算子分类**：
   - **Transformation**：`map`、`filter`、`flatMap`、`reduceByKey`、`join` 等（惰性执行）
   - **Action**：`collect`、`count`、`saveAsTextFile` 等（触发计算）
3. **深入 Shuffle 机制**：理解宽依赖的性能影响
4. **实践调优**：从实际业务场景中发现问题并优化

### 4.2 推荐实践
- **使用 Spark Shell 交互式学习**：快速验证算子行为

- **对比 DataFrame/Dataset**：理解 RDD 与高级 API 的差异与适用场景
- **关注官方文档**：Spark 版本迭代快，以官方文档为准

### 4.3 进阶方向
